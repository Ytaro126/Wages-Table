# admin-app

こちらはytworksappのユーザーを管理する管理者画面を作成する目的と手順書です。


1.管理者画面で実装したい機能。

・管理者画面でユーザーが入力した日付毎の配達個数、集荷の個数、売り上げ金を参照できるようにする。
・データの手動修正と、ユーザーの追加、停止ができるようにする。
・その日の売上が確定した際に管理者画面で確定登録をし、確定登録された日のデータはユーザー側で編集できないようにする。
・ログイン情報を管理する。
・月毎、年毎の集計を参照する。
・ユーザー側のアプリの異常を検知した際にアラートが表示されるようにする。
・管理者画面にログインできるのは1台のMacBookであると同時に自宅のWi-Fiからでないとアクセスできないようにする。

2.使用する人間と機器

・自分
・自分のMacBook

3.外観と機能

・ベンチマークはAWSコンソール。
・ヘッダーの左側にYTworksと表記。YTworksを押すとユーザー一覧に移る。
・すぐ下にハンバーガーメニューを表記、クリックすると左側にユーザー、ログイン履歴、アラート、ログアウトと表記されたメニューが左側から出現する。それぞれログアウト以外はボタンを押すとメニュー画面は固定されたままでメイン画面がクリックされたメニュー通りの画面になる。

4.メニューごとの役割

・ユーザー
登録されているユーザーのIDが表示される。対象のユーザーをクリックすると売上の各日付毎の配達個数や売上金が表示される。対象の日の右側に確定ボタンを押すとユーザーはその日のデータを編集できなくなる。

・ログイン履歴
ログイン、ログアウトしたユーザーが記録として残る。対象のログは31日で削除される。

・アラート
アプリが使用できない、データが消えるなどといった異常を知らせる。

・ログアウト
押すとログアウトしますか？のポップアップが表示され、はいを押すとログアウトする。



5.1~4を実装するために必要なAWSの機能

・Route 53
admin.ytworksapp.com 用のサブドメインを追加する。管理者画面は本体アプリとは分けて管理する。

・CloudFront
admin.ytworksapp.com の入口として使う。HTTPS 化と配信制御の起点にする。

・S3
管理者画面のフロントエンド一式を配置する。index.html、CSS、JavaScript を静的配信する。

・AWS Certificate Manager (ACM)
admin.ytworksapp.com 用の証明書を発行して CloudFront にアタッチする。

・Lambda
管理者画面用 API を作る。初期段階では以下の API を作る。
1. ユーザー一覧取得
2. ユーザー詳細取得
3. 日別データ取得
4. 売上確定登録
5. ログイン履歴取得
6. アラート一覧取得

・API Gateway
管理者画面フロントエンドから Lambda を呼び出すための API を公開する。

・DynamoDB または既存アプリ連携用 API
ログイン履歴、アラート、確定状態など管理者画面専用のデータ保存先を用意する。
ただし既存の本番データは SQLite を直接 Lambda から触らず、必要なら EC2 側の API 経由で参照・更新する。

・CloudWatch
Lambda エラーや API エラーをログとして確認する。アラート機能の土台にも使う。

・SNS
異常検知時の通知先として使う。まずはメール通知で十分。

・WAF は後回し
初期段階では必須ではない。まずは IP 制限、ベーシック認証、管理者ログインを優先する。

・IAM
管理者画面用の Lambda やデプロイ用ユーザーに必要最小限の権限を与える。

・IP 制限
admin.ytworksapp.com は自宅 Wi-Fi のグローバル IP のみ許可する。CloudFront だけで完結しにくい場合は API Gateway / Lambda URL ではなく、必要に応じて Nginx や別の入口構成も検討する。

・ベーシック認証
管理者画面の入口で追加の認証をかける。アプリ独自ログインの前にもう1段ガードを置く。

・1~4を実装するための具体的な AWS 作業

1. ユーザー画面
admin.ytworksapp.com を Route 53 と CloudFront で公開する。
ユーザー一覧取得 API、ユーザー詳細取得 API、日別売上取得 API を API Gateway + Lambda で用意する。
本番 SQLite を直接参照しない方針を決め、EC2 側 API 連携か管理用データ同期のどちらかに寄せる。

2. ログイン履歴
管理者ログイン成功/失敗、ログアウトを記録する保存先を DynamoDB に用意する。
31日で削除するために DynamoDB TTL を設定する。

3. アラート
CloudWatch Logs / CloudWatch Alarm で異常検知の元データを作る。
通知先として SNS を設定する。
管理者画面には Lambda 経由でアラート一覧を表示する。

4. ログアウト
アプリ側でログアウト処理を作る。AWS 側ではセッション保護のため API Gateway / Lambda の認証方式を決める。
将来的に Cognito を使うならここで採用を検討する。

実務寄りに考えると、最初から全部をサーバーレス化するより以下の順で進める。

第1段階
- admin.ytworksapp.com の公開
- ベーシック認証
- 自宅 Wi-Fi IP 制限
- 管理者画面フロントの土台

第2段階
- ユーザー一覧
- ユーザー詳細
- ログイン履歴

第3段階
- 売上確定
- アラート表示
- 管理者専用ログイン強化
